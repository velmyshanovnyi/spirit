## Знахідки

Немає. Перевірено конкретно:

- **Ризик виклику backup-кнопок у невалідному стані** (сховані елементи все одно приймають `.click()`): якщо профіль не створений, `state.identityKeyPair === null` → доступ до `.privateKey` кидає, але виняток ловиться `withBusyButton` і показується як статус — без падіння, без витоку секретів.
- **Ефемерний ключ (non-extractable)**: через нормальний UI backup-крок недосяжний (його відкриває лише `btn-profile-confirm`); навіть форсований клік дає лише caught-помилку.
- **Гігієна секретів**: `profile-passphrase` і `keyfile-passphrase` очищуються після використання; при помилці значення легітимно лишається для повтору.
- **Контракти викликів** збігаються з тестовими assertion-ами; HTML-фікстура тестів містить усі id, які чіпає імплементація, зі збігом початкових `hidden`-станів.
