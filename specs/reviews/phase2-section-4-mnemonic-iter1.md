## Контекст

Список слів BIP39 (2048 офіційних англійських слів) згенеровано через тимчасово встановлений npm-пакет `bip39` (перевірено: 2048 унікальних слів, перше "abandon", останнє "zoo"), записано у статичний дата-файл, пакет одразу видалено — жодної рантайм-залежності. Тест-вектор (entropy↔мнемоніка) також згенеровано незалежно через той самий пакет перед видаленням, для крос-перевірки з нашою реалізацією.

## Знахідки

Немає блокуючих. Підтверджено коректним:
- Алгоритм BIP39 (конкатенація entropy+checksum біт, 11-бітне групування, checksum = перші ENT/32 біт SHA-256(entropy)) — константи (32 байти, 8 біт checksum, 24 слова) взаємоузгоджені й відповідають офіційній таблиці BIP39 для 256-бітної ентропії.
- Файл списку слів — 2048 унікальних записів, усі проходять офіційні якірні перевірки.
- Хелпери конвертації байти↔біти — без off-by-one, коректно для байта 0.
- Усі заявлені шляхи помилок (невірна довжина ентропії, невірна кількість слів, слово поза списком, невірний checksum) досяжні й спрацьовують до будь-яких часткових обчислень; дублікати/переставлені слова ловляться через checksum.

**Реальна прогалина**: known-answer вектор шукає слова через ТОЙ САМИЙ файл списку, тож незалежно ловить помилки алгоритму, але не гарантовано ловить "неправильне, але внутрішньо узгоджене" спотворення самого файлу списку (одна помилка в слові на певному індексі, з якою погодилися б і вектор, і файл).

## Статус

Додано `bip39-wordlist.test.js` — незалежна перевірка цілісності файлу (довжина 2048, офіційні якорі, унікальність+lowercase-alphabetic, алфавітна відсортованість — справжня властивість офіційного списку). Перехід до ітерації 2.
